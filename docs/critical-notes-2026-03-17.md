# Critical Notes — 2026-03-17

Raw founder thoughts to address before launch:

1. **GP test over-filtering problem:** Super manual onboarding, 15-25 hand-picked scouts. Submission quality should be inherently higher. But we built a system that rejects 70-80% at the Thesis Quality Gate — is that appropriate for curated scouts? The heavy filtering makes sense for public S1 with unknown scouts, but for GP test with hand-picked people, we might be over-engineering the rejection pipeline and creating a frustrating experience.

2. **Token factory should be an early check:** Factory detection (Clanker, Bankr, etc.) should be one of the FIRST checks — it immediately saves us from rugs. Even with hand-picked scouts, we need this safety net. Consider moving factory check into Step 1 (structural filter) rather than Step 3.

3. **"Founder" dimension — should it be "Team" instead?** The current framework is founder-obsessed (weighted 2x). But many Base projects are team efforts. Should the dimension be reframed as "Team" to capture broader signal?

4. **Token focus area undefined:** What's our actual target? Low-cap? Mid-cap? High-cap? With token? Without? For the trade desk, the play needs to be ~10x on size. This feels like a serious gap — we haven't defined what kind of tokens we're actually looking for, which affects every scoring dimension.
